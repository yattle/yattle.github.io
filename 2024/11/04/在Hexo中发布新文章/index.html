2.注意事项

  • 清理生成的文件: 如果需要，可以在每次部署之前运行 hexo clean 来清理生成的静态文件。

3.完整流程总结

  1. 创建新文章：hexo new “文章标题”
  2. 编辑文章：在 source/_posts 中修改新文件。
  3. 生成静态文件：hexo generate
  4. 部署到 GitHub：hexo deploy
  5. 验证推送：在 GitHub 上查看新文章。
